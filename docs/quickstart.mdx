# 5 分钟开始使用

本页说明如何在较短时间内完成一次基于 GIS Agent 的应用构建、预览与分享。

前提条件：只需一个已注册的账号，以及一个你想要实现的想法。

## 这份文档适合回答哪些问题

如果你在问下面这些问题，这一页会比较适合先看：

- 如何开始使用 GIS Agent？
- GIS Agent 怎么用？
- 第一次使用时应该先做什么？
- 需求应该怎么描述？

## 步骤一：用自然语言描述需求

在 GIS Agent 的工作台对话框中，用自然语言清晰描述你的需求。

![](/docs/quickstart-assets/1.webp)

需求输入后，按回车或点击“开始会话”即可启动项目。

如果你不知道该怎么写需求，可以直接套用下面这个模板：

> 使用 [数据源]，完成 [操作或分析]，输出为 [页面、地图、报告或服务]。

例如：

> 使用武汉市行政区划和人口数据，生成分区统计地图，并输出为可分享的 Web 页面。

> 使用时空数据库中的 OSM 道路数据，分析某区域 15 分钟出行覆盖范围，并生成专题图。

你还可以通过以下方式发起需求：

1. 通过工具箱，从首页应用案例库中复制提示词。
2. 通过社区，在优秀应用中点击“做同款”快速复用设计思路。
3. 直接向 Agent 提问“你有什么技能？”，查看当前可用能力模块。

## 步骤二：生成与预览

GIS Agent 在接收到需求后，会自动分析需求并规划步骤，逐步生成应用代码。生成完成后，系统会自动构建并启动服务，并提供预览界面。

在此阶段，你可以：

- 预览应用，检查功能和布局是否符合预期。
- 直接在对话框中提出修改意见，由 Agent 持续调整结果。

![](/docs/quickstart-assets/2.webp)

## 步骤三：发布与分享

对预览结果满意后，点击页面右上角的“分享”按钮。

![](/docs/quickstart-assets/3.webp)

系统会自动完成部署并生成访问链接。部署成功后，你可以：

1. 复制访问链接，分享给他人使用。
2. 将应用发布到社区，供其他用户访问、点赞或通过“做同款”快速复刻。

![](/docs/quickstart-assets/4.webp)
