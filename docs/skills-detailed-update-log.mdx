## 二、新增技能能力

### 1. 3D Tiles CLI 技能

**来源**：`clis/3dtiles_cli`

**本轮新增能力**：
- 新建独立 3D Tiles 命令行工具技能入口，不再依赖零散脚本拼装
- 支持 3D Tiles 数据生成、导出、检查与执行计划输出
- 技能文档 `skills/3dtiles-cli/SKILL.md` 已建立，可直接作为代理技能入口
- 提供 Docker 构建脚本，便于在 sidecar 或独立运行时复用

**2026-06-25 增强点**：
- 优化矢量纹理导出默认值，减少手工传参
- 改进 `vector` 相关导出逻辑，增强默认材质与纹理生成行为
- 补充测试覆盖，保证导出默认行为可回归验证

### 2. 知识图谱技能

**来源**：`clis/knowledge_cli`

**本轮新增能力**：
- 强化知识图谱 G6 渲染模板，提升图谱结构展示与前端可读性
- 在 `skills/knowledge-cli/agents/openai.yaml` 中明确代理路由配置
- 增加 `validate_knowledge_graph_output.sh`，对知识图谱结果做结构校验

**本轮能力收敛方向**：
- 收紧技能触发规则，避免非知识库任务误走该技能
- 把技能说明从"能做什么"进一步收敛到"何时必须触发、何时禁止触发"

### 3. PostGIS 服务发布技能

**来源**：`clis/postgis_service_cli`

**本轮新增能力**：
- 新增 3D Tiles 发布能力，可从 Postgres / PostGIS 数据源暴露 3D Tiles 服务
- 新增 Cesium 预览链路，前端可直接预览 3D Tiles 数据集
- 增加 Postgres 3D Tiles 自动发现能力，减少手工登记数据源
- 增加大陆城市可重复导入流水线，形成样例数据与导入脚本闭环
- 增加大量 SQL、示例脚本与运行文档，补齐 3D Tiles 从数据准备到发布的操作面

**对技能层的直接意义**：
- `skills/postgis-service-cli/SKILL.md` 已从传统瓦片/样式指导扩展到 3D Tiles 发布与预览指导
- 技能调用时可引导代理：
  - 发现 3D Tiles 数据源
  - 发布服务
  - 打开预览
  - 处理样式、sprite、目录与 URL 输出

### 能力链路总览

1. `3dtiles-cli` 负责 3D Tiles 生成与导出
2. `postgis-service-cli` 负责 3D Tiles 发布、发现和预览
3. `knowledge-cli` 负责知识图谱输出与知识技能稳定路由

**结论**：这轮不只是"多了几个技能目录"，而是补齐了三条可被代理调用的能力面：
- 3D Tiles 生产能力
- 3D Tiles 服务发布与预览能力
- 知识图谱渲染与稳定触发能力
