# GIS Agent 是什么

**GIS Agent** 是面向行业场景的 GIS 智能生产平台。

它以自然语言交互为入口，把 GIS 数据处理、空间分析、地图制图、服务发布和成果整理连接起来，帮助用户更高效地完成空间任务。它不是传统意义上的聊天机器人，也不只是一个地图工具，而是一个能够理解需求、组织任务并生成成果的 GIS 工作助手。

> **一句话理解**
>
> 用对话方式组织 GIS 工作流，从提出需求到生成成果，减少中间的手工拆解与反复切换。

### 你可以用它做什么

- 直接描述一个空间任务，由系统组织数据接入、分析处理和成果输出。
- 支持专题制图、态势研判、选址分析、三维建模等常见任务。
- 可生成地图、页面、报告、服务链接以及可复用的流程成果。

---

## 产品定位

GIS Agent 的定位，不是单独替代某一个 GIS 功能模块，而是作为空间任务的统一入口与流程组织工具。

它更适合承担三类角色：

1. **任务入口**：用户先描述目标，再由系统组织后续流程。
2. **流程组织者**：把数据接入、分析处理、地图表达和成果交付串成连续链路。
3. **成果生成助手**：除说明和反馈外，还可以生成地图、页面、报告和服务。

与传统方式相比，主要变化体现在两点：

- **流程更集中**：传统模式下，数据处理、空间分析、制图表达、页面开发和服务发布往往分散在不同环节中，需要多人协作、逐步衔接。
- **衔接更顺畅**：在 GIS Agent 中，用户从描述需求开始，就可以推动数据接入、分析处理、地图表达、服务交付和成果复用依次向前运行。

这意味着 GIS 不再只是一个需要逐步操作的专业软件体系，也可以更直接地服务于任务交付。

---

## 核心问题

常见问题主要集中在以下几个方面。

- **交付周期长**：从需求提出到成果形成，通常需要经过数据准备、分析处理、制图表达、页面开发和服务发布等多个环节，整体周期往往较长。
- **专业门槛高**：很多任务依赖专业 GIS 软件操作、空间分析经验和一定的开发能力，业务人员通常难以直接参与。
- **成果难沉淀**：很多成果依赖个人经验完成，后续不容易沉淀为可复用的模板、流程或方法。

---

## 产品价值

针对上述问题，GIS Agent 主要提供以下支持。

- **缩短交付周期**：通过减少重复操作和中间切换，帮助缩短从需求到结果的处理时间。
- **降低使用门槛**：用户可以直接通过自然语言描述任务，减少对专业 GIS 操作环节的依赖。
- **提升成果质量**：在分析、制图和发布环节提供连续支持，输出结果可以覆盖地图、报告、页面和标准服务等形式。
- **沉淀组织能力**：可将相对成熟的处理流程沉淀为 Skill、模板或其他可复用成果，便于后续复用与推广。

---

## 与通用 AI、传统 GIS 的区别

GIS Agent 不是替代 GIS，也不是替代大模型，而是在两者之间建立更直接的协作关系，用于支持空间任务的执行与交付。

| 对比维度 | 通用 AI 工具 | 传统 GIS 平台 | GIS Agent |
| --- | --- | --- | --- |
| 输入方式 | 自然语言问答 | 专业软件操作 | 自然语言 + GIS 专业能力调度 |
| 空间分析能力 | 通常不具备专业执行能力 | 具备，但依赖人工操作 | 具备，并可自动执行流程 |
| 输出结果 | 文本、代码、建议 | 地图、数据、服务 | 地图、报告、页面、服务、Skill |
| 业务响应速度 | 快速回答，但难直接交付 | 能交付，但流程较长 | 在响应速度与交付结果之间取得平衡 |
| 经验复用方式 | 依赖提示词经验 | 依赖人工模板和脚本 | 可封装为可复用 Skill |
| 既有资产兼容 | 一般 | 强 | 强，可与既有 GIS 资源和系统协同使用 |

## 核心能力

如果把 GIS Agent 看作一条工作链路，它的能力大致可以概括为以下六个部分。

GIS Agent 覆盖了“需求理解 → 数据接入 → 空间分析 → 制图表达 → 服务发布 → 成果复用”的主要环节。

- **智能数据接入**：支持文件、数据库、API、在线地图服务等多源数据接入。
- **空间分析与计算**：支持缓冲区分析、裁剪、叠加、流域提取、三维地质建模等专业任务。
- **地图制图与可视化**：支持专题图、二维可视化、三维展示、交互式大屏等成果形态。
- **服务发布与交付**：支持 HTML 页面、标准地图服务、服务 URL 等成果交付。
- **成果导出与管理**：支持图片、矢量数据、报告文档等多种输出形式。
- **工作流沉淀复用**：支持将成熟流程封装为可持续复用的 Skill。

这意味着它不仅适用于单次任务处理，也适合把常见流程逐步沉淀为可复用的工作方式。

## 典型业务场景

下面这些场景较能体现 GIS Agent 的使用方式。

### 全时空智能制图

- **适用场景**：专题制图、底图风格统一、图例配置、中文标注、成果微调等地图表达类任务。
- **常见难点**：既要处理 GIS 数据和表达逻辑，又要兼顾配色、排版和标注规则，修改时容易重复返工。
- **处理方式**：通过对话描述表达目标、底图风格和展示要求，由系统完成理解、执行与微调。
- **示例任务**：`请将 OSM 数据入库到空间数据库，采用高德电子地图风格显示，增加图例和中文标注。`

### 防汛应急态势研判

- **适用场景**：整合降雨、水位、排涝设施、风险点等多源数据，形成态势图、风险等级图、分析报告和服务链接。
- **常见难点**：数据来源多、时效要求高，传统方式通常依赖人工整理和反复出图。
- **处理方式**：通过自然语言描述研判目标和展示要求，由系统组织数据接入、风险分级、地图表达和成果输出。
- **示例任务**：`整合实时降雨、河道水位、排涝设施与风险点数据，生成全市防汛综合态势图，并按街道进行风险等级着色。`

### 商业选址与客流分析

- **适用场景**：围绕 POI、交通、人口和竞争分布开展商圈分析、选址评估和客流预测。
- **常见难点**：通常需要整合多类空间数据，分析流程较长，结果反馈不够及时。
- **处理方式**：通过描述评估范围、分析对象和目标结果，由系统完成商圈分析、密度表达和结果汇总。
- **示例任务**：`分析规划地铁站周边 800 米步行范围内现有商业网点密度、业态结构，并预测未来客流潜力。`

### 三维地质建模与工程勘察

- **适用场景**：基于钻孔数据构建三维地质模型，并进入展示、评审和成果交付环节。
- **常见难点**：专业性强、流程复杂，成果从分析到展示通常依赖资深人员长期参与。
- **处理方式**：通过描述建模目标和展示需求，由系统协助完成模型构建、成果表达和交互式页面生成。
- **示例任务**：`基于钻孔数据自动构建三维地质模型，并生成可交互展示页面。`

## 成果交付

GIS Agent 的输出不局限于单一页面或单次结果，而是可以根据不同交付场景形成多种成果。

- **地图成果**：专题地图、热力图、二维可视化、三维态势图、可交互大屏。
- **数据成果**：GeoJSON、SHP、矢量瓦片、栅格瓦片等标准化空间数据成果。
- **文档成果**：PDF、Word、Markdown 报告与分析记录。
- **服务成果**：标准地图服务、服务 URL、HTML 页面、可在线访问的交互式应用。
- **能力成果**：可重复调用的 Skill、案例模板、流程资产。

## 优先使用场景

下面这些类型通常更适合使用 GIS Agent。

- **智能制图**：专题制图、风格统一、图例配置、中文标注、成果微调等任务。
- **应急管理**：防汛、防灾、风险预警、资源调度等任务。
- **规划分析**：选址、商圈分析、交通影响评估、区域画像等任务。
- **展示交付**：数字孪生展示、专题大屏、三维模型展示等任务。
