# 高频问题与解答

本文整理了 GIS Agent 在产品认知、数据使用、成果管理和技术排查方面的常见问题。

## 产品认知与定位

### Q1: GIS Agent 是什么？它与通用 AI 工具的核心区别在哪里？

**A:** GIS Agent 并非通用聊天机器人，其定位是 **“全时空智能生产操作系统”** 与 **“空间智能副驾驶”**。

* **核心区别：** 通用 AI（如 ChatGPT）擅长处理文本、代码等通用信息，但无法理解地理空间概念与操作。GIS Agent 深度集成于 **GIS 信创生态**，可直接理解并执行“缓冲区分析”、“流域提取”、“三维地质建模”等专业 GIS 任务。
* **本质差异：** 通用 AI 更偏向“问答式”信息助手；GIS Agent 更偏向“交付式”专业团队，可直接产出地图应用与空间分析报告。

### Q2: GIS Agent 旨在解决哪些核心痛点？

**A:** 聚焦三大痛点：

1. **降低使用门槛：** 传统 GIS 软件操作复杂，学习**成本高。GIS Agent 支持** **自然语言描述需求**，用户无需掌握底层库、数据库或后端开发技术。
2. **提升响应效率：** 传统空间分析需求从提出到交付常需数天至数周。GIS Agent 可将流程压缩至分钟级或小时级，效率提升超 10 倍。
3. **沉淀数字资产：** 项目经验易随人员流动而流失。GIS Agent 可将成功交付成果沉淀为可复用的行业 Skill，实现经验的数字化与规模化复用。

### Q3: 产品名称是“GIS Agent”还是“GIS GeoAgent”？

**A:** 内部全称为 **GIS GeoAgent**，对外发布及日常沟通中，统一使用 **GIS Agent** 这一简洁称谓，两者指向同一产品。

## 使用基础与数据

### Q4: 使用 GIS Agent 是否需要自行准备 API Key 或大模型？

**A: 不需要**。

* **在线版 (SaaS):** 开箱即用，用户登录网页即可使用，系统自动调度内置模型。
* **本地部署版:** 由部署单位提供模型资源，用户无需自行申请与管理 API Key。

### Q5: GIS Agent 在处理中国地理数据方面有何优势？

**A:** 核心优势在于 **本地化适配 与 自主可控**。

* **坐标系支持：** 内置 GIS 时空数据库内核，精准支持 **CGCS2000、北京 54、西安 80** 等国内常用坐标系。
* **数据理解：** 深度理解中文地理命名规范与行政区划体系。
* **安全合规：** 完全遵循信创标准，确保国内敏感数据处理的安全性与合规性。

### Q6: GIS Agent 中的数据从哪里来？

**A:** GIS Agent 本身不预存空间数据。其数据来源于：

1. **用户上传：** 支持上传 Shapefile、GeoJSON、CSV 等格式的本地数据。
2. **指定数据库：** 可连接并调用用户指定的新一代时空数据库中的数据（例如 OSM 数据）。
3. **在线服务：** 支持接入合规的在线地图服务（WMS/WFS）作为数据源。

**操作建议：** 在对话中明确指定数据源，例如：“使用当前数据库中的 OSM 数据进行缓冲区分析”。

### Q7: 能否对现有数据进行范围裁剪或合并操作？

**A:** 可以。这是处理大规模数据（如全量 OSM）以提升性能的常见操作。

* **操作逻辑：** 通过新一代时空数据库对数据按行政区划或其他范围进行裁剪或合并，再将结果发布为矢量瓦片服务（Vector Tile），可大幅提升前端加载速度与业务贴合度。
* **提示词参考：** “针对数据库中的 OSM 数据，按‘北京市’行政区划进行裁剪，并发布为矢量瓦片服务。”

## 成果输出与管理

### Q8: 使用 GIS Agent 生成的分析成果（地图、报告）保存在哪里？

**A:** 保存位置取决于使用模式：

* **在线版 (SaaS):** 所有成果保存在用户的个人云端项目空间，可通过“我的作品”或“项目列表”进行查看、管理、分享与再编辑。
* **本地部署版:** 所有成果保存在单位内部服务器，数据完全由单位掌控，安全性最高。

### Q9: 生成的报告或地图支持导出哪些格式？可以导出图片吗？

**A:** 支持多种导出格式，满足不同场景需求。

* **地图/可视化成果:**
  * 图片格式：**PNG、JPG**，便于分享与汇报。
  * 矢量格式：**GeoJSON、SHP**，供其他 GIS 软件编辑。
  * 服务发布：可直接发布为 **Web 地图服务 (WMS/WFS)** 供在线调用。
* **分析报告:** 支持导出为 **PDF、Word、Markdown** 格式，便于存档、打印与二次编辑。

### Q10: 如何处理地方独立坐标系或坐标不匹配的数据？

**A:** GIS Agent 具备强大的坐标处理与数据融合能力。

* **自动处理：** 当上传数据坐标不统一时，可通过自然语言指令触发系统的自动识别、匹配与转换功能。
  * **指令示例：** “将我上传的‘地块.shp’（北京 54 坐标系）投影转换到 WGS84 坐标系，并与‘POI数据.csv’（WGS84）进行叠加分析。”
* **高级配置：** 对于复杂的自定义坐标系，建议在专业版中通过参数化界面进行精确配置。

## 技术问题排查指南

> **说明：** 本节为常见技术问题的系统性排查指引，应与通用 FAQ 区分，提供具体操作路径。

### **Q11: 页面预览报错或功能异常**

**A:** 请按以下优先级尝试修复：

1. **AI 自修复（推荐）：** 将问题现象直接告知 Agent，让其分析日志并尝试修复。
2. **切换模型重试：** 不同模型能力存在差异，可尝试切换至 Qwen3.6-Plus 或 Kimi 2.5 等模型再次执行任务。
3. **提供详细错误日志：** 打开应用日志或浏览器控制台（F12），复制红色 JS Error 报错信息反馈给 AI 以引导精准修复。
4. **版本回退：** 如果修改导致问题，直接回退到上一个可用版本通常比反复调试更高效。

### **Q12: 成果已生成但页面不显示**

**A:** 此问题多为前端渲染或数据加载异常。

* **操作步骤：**
  1. 按 **F12** 打开浏览器开发者工具。
  2. 切换到 **控制台 (Console)** 标签页。
  3. 复制其中的报错信息（通常是红色错误日志）。
  4. 将错误日志反馈给 Agent，指令其针对性修复（例如：“控制台报错‘XXX’，请修复前端瓦片加载问题”）。
* **解析：** 此方法能直接定位到渲染异常、瓦片加载失败或跨域（CORS）等问题的根源。

### **Q13: 地图或界面出现字体错误/乱码**

**A:** 地图前端渲染依赖特定字体文件，缺失会导致乱码。

* **快速解决方案：** 在对话中补充以下提示词，引入公共字体库。

```
使用字体 https://fonts.loli.net/css?family=Open+Sans
```

* **解析：** 该指令会引导前端代码加载 Open Sans 字体，通常可快速解决字体缺失导致的显示问题。

### **Q14: 任务执行时对话栏转圈或卡住不动**

**A:** 需区分情况处理。

* **仅对话栏转圈：** 此多为前端显示延迟，**任务实际已在后端执行完成**，不影响后续操作，可忽略并继续使用。
* **项目页面无法加载：** 请依次尝试：
  1. **刷新页面。**
  2. 在对话框输入 `/` 唤起工具栏，点击 **重启应用**。
  3. 引导 AI 分析修复，例如输入：“项目未成功启动，目前无法预览，请分析项目构建或启动问题并修复”。

### **Q15: 网站卡死或无响应**

**A:** 通常由大模型会话上下文过长或前端渲染数据量过大导致。

* **直接处理：** **刷新页面**，然后重新发起对话。
* **预防建议：** 后续操作中，通过缩小分析数据的地理范围或复杂度，降低页面负载，减少卡死概率。

***

**使用提示：** 若遇到未涵盖的问题，建议在对话中清晰描述现象，包括数据情况、操作步骤和错误提示，Agent 将为您提供定制化解决方案。
